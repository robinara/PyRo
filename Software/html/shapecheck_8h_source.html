26  * The function areShapesConformable(A,B) checks that the shapes
27  * A and B are conformable (i.e. the same size/geometry). Typically
28  * the A and B parameters are of type TinyVector<int,N_rank> and represent
29  * the extent of the arrays. It's possible that in the future jagged-edged
30  * arrays will be supported, in which case shapes may be lists
31  * of subdomains.
32  */

36 {
37  // If the shape objects are different types, this means
38  // that the arrays are different ranks, or one is jagged
