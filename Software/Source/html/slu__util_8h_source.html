125 /*
126  *-- This contains the options used to control the solution process.
127  *
128  * Fact (fact_t)
129  * Specifies whether or not the factored form of the matrix
130  * A is supplied on entry, and if not, how the matrix A should
131  * be factorizaed.
132  * = DOFACT: The matrix A will be factorized from scratch, and the
133  * factors will be stored in L and U.
134  * = SamePattern: The matrix A will be factorized assuming
135  * that a factorization of a matrix with the same sparsity
136  * pattern was performed prior to this one. Therefore, this
137  * factorization will reuse column permutation vector
138  * ScalePermstruct->perm_c and the column elimination tree
139  * LUstruct->etree.
140  * = SamePattern_SameRowPerm: The matrix A will be factorized
141  * assuming that a factorization of a matrix with the same
142  * sparsity pattern and similar numerical values was performed
143  * prior to this one. Therefore, this factorization will reuse
144  * both row and column scaling factors R and C, both row and
145  * column permutation vectors perm_r and perm_c, and the
146  * data structure set up from the previous symbolic factorization.
147  * = FACTORED: On entry, L, U, perm_r and perm_c contain the
148  * factored form of A. If DiagScale is not NOEQUIL, the matrix
149  * A has been equilibrated with scaling factors R and C.
150  *
151  * Equil (yes_no_t)
152  * Specifies whether to equilibrate the system (scale A's row and
153  * columns to have unit norm).
154  *
155  * ColPerm (colperm_t)
156  * Specifies what type of column permutation to use to reduce fill.
157  * = NATURAL: use the natural ordering
158  * = MMD_ATA: use minimum degree ordering on structure of A'*A
159  * = MMD_AT_PLUS_A: use minimum degree ordering on structure of A'+A
160  * = COLAMD: use approximate minimum degree column ordering
161  * = MY_PERMC: use the ordering specified by the user
162  *
163  * Trans (trans_t)
164  * Specifies the form of the system of equations:
165  * = NOTRANS: A * X = B (No transpose)
166  * = TRANS: A**T * X = B (Transpose)
167  * = CONJ: A**H * X = B (Transpose)
168  *
169  * IterRefine (IterRefine_t)
170  * Specifies whether to perform iterative refinement.
171  * = NO: no iterative refinement
172  * = SLU_SINGLE: perform iterative refinement in single precision
173  * = SLU_DOUBLE: perform iterative refinement in double precision
174  * = SLU_EXTRA: perform iterative refinement in extra precision
175  *
176  * DiagPivotThresh (double, in [0.0, 1.0]) (only for sequential SuperLU)
177  * Specifies the threshold used for a diagonal entry to be an
178  * acceptable pivot.
179  *
180  * SymmetricMode (yest_no_t)
181  * Specifies whether to use symmetric mode. Symmetric mode gives
182  * preference to diagonal pivots, and uses an (A'+A)-based column
183  * permutation algorithm.
184  *
185  * PivotGrowth (yes_no_t)
186  * Specifies whether to compute the reciprocal pivot growth.
187  *
188  * ConditionNumber (ues_no_t)
189  * Specifies whether to compute the reciprocal condition number.
190  *
191  * RowPerm (rowperm_t) (only for SuperLU_DIST or ILU)
192  * Specifies whether to permute rows of the original matrix.
193  * = NO: not to permute the rows
194  * = LargeDiag: make the diagonal large relative to the off-diagonal
195  * = MY_PERMR: use the permutation given by the user
196  *
197  * ILU_DropRule (int)
198  * Specifies the dropping rule:
199  * = DROP_BASIC: Basic dropping rule, supernodal based ILUTP(tau).
200  * = DROP_PROWS: Supernodal based ILUTP(p,tau), p = gamma * nnz(A)/n.
201  * = DROP_COLUMN: Variant of ILUTP(p,tau), for j-th column,
202  * p = gamma * nnz(A(:,j)).
203  * = DROP_AREA: Variation of ILUTP, for j-th column, use
204  * nnz(F(:,1:j)) / nnz(A(:,1:j)) to control memory.
205  * = DROP_DYNAMIC: Modify the threshold tau during factorizaion:
206  * If nnz(L(:,1:j)) / nnz(A(:,1:j)) > gamma
207  * tau_L(j) := MIN(tau_0, tau_L(j-1) * 2);
208  * Otherwise
209  * tau_L(j) := MAX(tau_0, tau_L(j-1) / 2);
210  * tau_U(j) uses the similar rule.
211  * NOTE: the thresholds used by L and U are separate.
212  * = DROP_INTERP: Compute the second dropping threshold by
213  * interpolation instead of sorting (default).
214  * In this case, the actual fill ratio is not
215  * guaranteed to be smaller than gamma.
216  * Note: DROP_PROWS, DROP_COLUMN and DROP_AREA are mutually exclusive.
217  * ( Default: DROP_BASIC | DROP_AREA )
218  *
219  * ILU_DropTol (double)
220  * numerical threshold for dropping.
221  *
222  * ILU_FillFactor (double)
223  * Gamma in the secondary dropping.
224  *
225  * ILU_Norm (norm_t)
226  * Specify which norm to use to measure the row size in a
227  * supernode: infinity-norm, 1-norm, or 2-norm.
228  *
229  * ILU_FillTol (double)
230  * numerical threshold for zero pivot perturbation.
231  *
232  * ILU_MILU (milu_t)
233  * Specifies which version of MILU to use.
234  *
235  * ILU_MILU_Dim (double)
236  * Dimension of the PDE if available.
237  *
238  * ReplaceTinyPivot (yes_no_t) (only for SuperLU_DIST)
239  * Specifies whether to replace the tiny diagonals by
240  * sqrt(epsilon)*||A|| during LU factorization.
241  *
242  * SolveInitialized (yes_no_t) (only for SuperLU_DIST)
243  * Specifies whether the initialization has been performed to the
244  * triangular solve.
245  *
246  * RefineInitialized (yes_no_t) (only for SuperLU_DIST)
247  * Specifies whether the initialization has been performed to the
248  * sparse matrix-vector multiplication routine needed in iterative
249  * refinement.
250  *
251  * PrintStat (yes_no_t)
252  * Specifies whether to print the solver's statistics.
253  */
